OpenClaw 完整配置

从零开始配置 OpenClaw 交易辅助系统，包括安装、Provider 配置、Skills 安装和 API Keys 管理。

📌 简介

OpenClaw 是 AI 助手执行框架，支持技能扩展、自动化工具调用和多平台消息集成。本教程从零开始配置完整的交易辅助系统。

⏱️ 预计时间：30-60 分钟，取决于网络环境和 API 申请进度。
入门必备环境配置技能安装

🔧 步骤 1：安装 OpenClaw

系统要求

  • Node.js v18+（推荐 v20+）
  • npm 或 pnpm 包管理器
  • Git（用于技能管理）

安装命令

# 全局安装 OpenClaw
npm install -g openclaw

# 初始化配置
openclaw onboard

验证安装

openclaw status

应显示 OpenClaw 版本、当前配置和连接状态。

⚙️ 步骤 2：配置 Provider

编辑 ~/.openclaw/openclaw.json，添加模型 Provider：

{
  "models": {
    "providers": {
      "bailian": {
        "baseUrl": "https://coding.dashscope.aliyuncs.com/v1",
        "apiKey": "sk-your-api-key",
        "api": "openai-completions"
      }
    }
  }
}

获取 API Key

  • 访问 阿里云百炼控制台
  • 注册/登录账号
  • 创建 API Key
  • 复制 Key 到配置文件
💡 推荐模型：qwen-plus（平衡性能与成本），qwen-max（复杂任务）。

📦 步骤 3：安装 Skills

根据交易需求安装相关技能：

核心交易技能

# Simmer 交易接口（Polymarket/Kalshi）
npx skills add joeseesun/simmer -y

# 网页内容提取
npx skills add joeseesun/defuddle-skill -y

辅助技能（可选）

# 设计顾问
npx skills add joeseesun/qiaomu-design-advisor -y

# 学习网站生成
npx skills add joeseesun/knowledge-site-creator -y

# NotebookLM 集成
npx skills add joeseesun/anything-to-notebooklm -y

验证技能列表

npx skills list

🔑 步骤 4：配置 API Keys

⚠️ 安全提示：不要将 API keys 提交到 Git 仓库！使用 .env 文件或环境变量管理。

方式 A：环境变量（推荐）

# 添加到 ~/.bashrc 或 ~/.zshrc
export SIMMER_API_KEY="sk_live_..."
export MEMOS_API_KEY="..."
export DASHSCOPE_API_KEY="sk-..."

# 重新加载配置
source ~/.bashrc  # 或 source ~/.zshrc

方式 B：.env 文件

# 在 ~/.openclaw/.env 文件中
SIMMER_API_KEY=sk_live_...
MEMOS_API_KEY=...
DASHSCOPE_API_KEY=sk-...

必需 Keys 清单

  • SIMMER_API_KEY：Simmer 交易接口（注册 simmer.markets 获取）
  • DASHSCOPE_API_KEY：阿里云百炼模型 API
  • 其他：根据安装的技能要求配置

✅ 验证安装

检查 OpenClaw 状态

openclaw status

检查技能列表

npx skills list

测试模型连接

openclaw run "Hello, World!"

测试 Simmer 连接（如已配置）

curl https://api.simmer.markets/api/sdk/agents/me \
  -H "Authorization: Bearer $SIMMER_API_KEY"

🔧 常见问题

Q: npm install 失败？

  • 检查 Node.js 版本：node -v（需 v18+）
  • 尝试清理缓存：npm cache clean --force
  • 使用国内镜像：npm config set registry https://registry.npmmirror.com

Q: openclaw status 显示 Provider 错误？

  • 检查 API Key 是否正确复制（无多余空格）
  • 确认 API Key 有足够额度
  • 检查网络连接（可能需要代理）

Q: Skills 安装失败？

  • 确认 Git 已安装：git --version
  • 检查技能仓库地址是否正确
  • 查看错误日志，可能是依赖冲突
