# PRD - Aegis Arbitrage (AI 全自动化套利与投资系统)

## 1. 项目概述
Aegis Arbitrage 是一个面向预测市场（优先 Polymarket）的 AI 自动化套利与投资系统，目标是在可控风险下实现稳定收益，并支持人工一键接管。

## 2. 项目目标（V1）
1) 自动发现套利机会（跨市场/跨平台/时序错位）
2) 自动执行交易（下单/撤单/重试）
3) 内置硬风控（单笔限额、日损、熔断）
4) 自动产出日报/周报（收益、回撤、策略表现）

## 3. 非目标（V1 不做）
- 全资产类别覆盖（先聚焦 Polymarket + 核心外部价格源）
- 全自动高频做市（先做策略驱动套利）
- 无监督自学习实盘（先人工审核参数变更）

## 4. 用户与使用场景
- Owner: Robin
- Operators: Jarvis（策略与信息）、Zorro（系统与执行）
- 场景：每日自动扫描 → 自动执行 → 自动汇报；异常时人工接管。

## 5. 功能需求
### 5.1 数据层
- Polymarket 市场、盘口、成交数据接入
- 外部价格源（交易所/指数）
- 社媒与新闻事件信号（由 Jarvis 汇总）

### 5.2 信号层
- YES+NO<1 套利扫描
- 跨相关市场错配检测
- 延迟套利窗口检测（短周期市场）

### 5.3 决策层
- 策略优先级排序（收益/风险/置信度）
- 仓位分配（风险预算）
- 执行门槛控制（最小 edge、最小流动性）

### 5.4 执行层
- 自动下单、撤单、重试
- 成交确认与状态机
- 故障降级（半自动模式）

### 5.5 风控层（强制）
- 单笔最大风险敞口
- 单日最大亏损阈值
- 最大回撤阈值
- 熔断：API 异常/滑点异常/波动异常

### 5.6 报告层
- 日报：机会数、成交数、PnL、胜率、异常
- 周报：策略分解、回撤分析、下周优化建议

## 6. 技术需求
- 可在 WSL2 稳定运行（Zorro）
- 低延迟数据处理与异步执行
- 完整日志与审计追踪
- 配置可版本化（GitHub）

## 7. KPI（V1）
- 策略执行可用率 ≥ 99%
- 风控触发准确率（无漏触发）
- 日报自动生成成功率 ≥ 95%
- 周回撤控制在预算阈值内

## 8. 里程碑
- M1（Week1）：数据接入 + 扫描器 + 风控骨架
- M2（Week2）：自动执行 + 报告自动化
- M3（Week3）：多策略并行 + 参数管理
- M4（Week4）：稳定化 + 复盘闭环

## 9. 风险与缓解
- 数据延迟：多源校验 + 时间戳对齐
- 滑点风险：执行前流动性检查 + 价格保护
- 黑天鹅：熔断 + 手动接管
- 过拟合：回测分层 + 小资金灰度上线

## 10. 协作分工（方案A）
- Jarvis：需求拆解、信号输入、策略文档、报告审核
- Zorro：系统开发、执行引擎、风控实现、部署运维
