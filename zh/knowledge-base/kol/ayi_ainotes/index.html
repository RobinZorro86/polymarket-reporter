为什么值得学：这类页面帮助你理解“如何组织系统”，而不是只知道“下一单买什么”。

🔄 Workflow 架构

设计原则

  1. 模块化：技能独立、可插拔
  2. 可观测：日志完整，便于调试与回溯
  3. 风控优先：止损与熔断前置
  4. 渐进式：虚拟盘 → 小额实盘 → 扩大规模

推荐技能组合

  • Simmer SDK：执行核心，支持虚拟盘
  • Defuddle：内容提取与新闻处理
  • Tavily Search：发现机会与信号
  • Voice Command：可选控制层

⚙️ 配置建议

训练模式

  • 先用 Simmer 虚拟盘训练
  • 胜率稳定到一定水平后再考虑实盘
  • 每笔交易记录逻辑与结果

实盘模式

  • 单笔限额：$100（Simmer 默认）
  • 每日限额：$500
  • 连接外部钱包（MetaMask + Polygon）
  • 启用 auto_redeem
1. 扫描机会
2. 验证信号
3. 风险评估
4. 执行交易
5. 记录日志
6. 定期复盘
风险提示：不要把“工作流自动化”误解成“可以不理解策略”。自动化只会放大你的方法质量。
