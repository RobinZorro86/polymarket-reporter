BTC 5 分钟市场在剧烈波动后已重新定价 - BTC 15 分钟市场尚未重新定价 - 相邻的 5 分钟市场也处于滞后状态 → 贝叶斯层帮助确定哪个市场现在滞后于更现实的概率估计，成为套利的第一切入点。 --- ## 模型 2：Edge（优势模型） ### 核心作用 判断检测到的错位是否代表**真实的数学优势**，而不仅仅是可见的价格差异。 ### 为什么重要 套利机器人只应进入那些在扣除所有成本后仍存在**正预期价值 (EV)** 的交易。否则，它只是在交易噪音，并烧掉手续费资本。 ### 公式 **单一市场内**： ``` 如果 p_yes + p_no < 1（两边可以用总和小于 1 的价格买入） Edge = 1 - (p_yes + p_no) - c 例如：两边之和为 0.95，总成本 0.02 → Edge = 1 - 0.95 - 0.02 = 0.03（净优势 3%） ``` **相关市场之间**： ``` EV = q - p EV_net = q - p - c q = 模型内部概率 p = 市场价格 c = 手续费 + 点差 + 滑点 + 不完全执行风险 ``` > **Edge 是整个系统的主要过滤器。没有它，机器人就不是在做套利，而只是在制造不必要的活动。** --- ## 模型 3：Spread（价差模型） ### 核心作用 识别**跨市场错位**——两个相关市场暂时偏离超过正常水平的情况。 ### 为什么重要 对于 Polymarket 5 分钟加密市场尤其有意义，因为： - 5 分钟和 15 分钟市场可能存在延迟同步 - 相邻 5 分钟窗口可能不均匀更新 - 一个合约可能被高估，而另一个被低估 ### 公式 ``` S = P1 - P2 （当前价差） z = (S - μ_S) / σ_S （Z 分数 = 价差偏离正常的程度） μ_S = 正常平均价差 σ_S = 价差标准差 ``` ### 机器人做法 构建相关市场地图： - BTC 5m vs BTC 15m - 当前 BTC 5m 窗口 vs 下一个 BTC 5m 窗口 - 单个市场 vs 相关概率的合成组合 如果当前价差 Z 分数远超正常范围 → 套利机会。 --- ## 模型 4：Stoikov（执行质量模型） ### 核心作用 **高质量执行套利结构**——这是套利中最难的部分。 ### 为什么重要 在套利中，最难的问题往往不是发现错位，而是及时组装结构的两部分而不损失优势。 ### 关键公式 ``` r = s - q × γ × σ² × (T - t) r = 内部报价价格 s = 中间价 q = 当前库存 γ = 风险厌恶系数 σ² = 方差 (T - t) = 剩余时间 ``` ### 含义 库存失衡越大、风险越高，就越需要调整执行价格。 ### Stoikov 层让机器人做什么 - 不再机械地增加已积累的一侧仓位 - 更积极地完成缺失的部分 - 根据当前库存失衡重新定位限价单 - 决定何时保持被动、何时切换到更激进的执行 > **Stoikov 确保机器人不仅「发现」套利，还能高效「抓住」它。** --- ## 模型 5：Kelly（凯利准则） ### 核心作用 **选择仓位大小**——即使发现很强的套利机会，也不意味着全部资金押上去。 ### 为什么重要 5 分钟市场始终存在操作风险： - 结构第二部分可能无法成交 - 流动性可能消失 - 成本可能上升 - 优势可能被高估 - 多个类似交易可能同时开启 ### 公式 **经典 Kelly**： ``` f* = (b × p - q) / b f* = 最优资金比例 b = 每单位风险的回报 p = 成功概率 q = 1 - p ``` **实际使用**：分数 Kelly（更保守）： ``` f = λ × f*（λ < 1） 例如：0.5 Kelly 或 0.25 Kelly ``` ### Kelly 层根据什么决定资金分配 - 优势大小 - 完全执行的概率 - 订单簿深度 - 错位收敛的速度 - 其他仓位已承诺的总资金 --- ## 模型 6：Monte Carlo（蒙特卡洛模拟） ### 核心作用 **测试整个系统在真实市场中是否真正可行**——防止自我欺骗的保护层。 ### 为什么重要 纸面上，几乎任何套利机器人看起来都很诱人：有优势、有价差、有清晰信号。 但真实市场更艰难： - 结构第二部分仅部分成交 - 价差比预期提前收敛 - 流动性消失 - 手续费更高 - 一连串糟糕执行导致回撤 ### 模拟参数 将策略运行数千次不同情景： - 不同成交率 - 不同滑点水平 - 随机执行延迟 - 不同比例的不完整结构 - 概率估计误差 - 连续失败交易集群 ### 模拟输出 - 真实预期增长 - 可能回撤 - 策略在何处失去稳健性 - 实际安全的仓位大小 > **Monte Carlo 是防止自我欺骗的保护层。** --- ## 六个模型的组装逻辑 ``` Bayesian → 在5分钟及相关市场中估算公平概率 ↓ Edge → 检查扣除成本后是否仍有真实正EV ↓ Spread → 识别相关合约间的错位 ↓ Stoikov → 高效完成结构两部分而不损失执行优势 ↓ Kelly → 决定仓位大小 ↓ Monte Carlo → 测试整个设置能否在真实市场而非干净回测中生存 ``` --- ## 对普通人的意义 ### 如果你是技术型用户 这套框架可以直接作为搭建自己量化系统的参考： - 6 个模块可以独立开发 - 每个模块有清晰的输入输出 - 可以逐步叠加验证 ### 如果你是普通用户 了解这 6 个模型可以帮助你： - **理解为什么 Bot 能赚钱**：不是靠预测，而是靠系统化的数学优势 - **理解 LP vs 套利的区别**：LP 是被动收入，套利是主动系统 - **评估跟单的风险**：套利系统也有容量限制和失效可能 ### 最值得记住的一点 > **没有 Edge 过滤的套利，不是在套利，而是在制造不必要的活动。** --- ## 作者其他教程资源 | 教程 | 链接 | |------|------| | Polymarket 全网最全新手教程 | [2022702972359266612](https://x.com/lqp2021/status/2032702972359266612) | | PolyCop 保姆级跟单交易教程 | [2033423660338618546](https://x.com/lqp2021/status/2033423660338618546) | | 用 OpenClaw 实现 Polymarket 比特币 5/15 分钟自动化交易系统 | [2033790500604682425](https://x.com/lqp2021/status/2033790500604682425) | --- **整理者**: Zorro-Research 🔍 **用途**: 量化套利 / 数学模型 / 系统架构 **版本**: 1.0 **整理日期**: 2026-03-27
