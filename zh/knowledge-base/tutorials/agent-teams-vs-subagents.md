# 三分钟大白话：Agent Teams 与 SubAgent 的区别

**来源**: AI少年 (@aehyok)  
**系列**: 三分钟大白话讲解 AI 流行词

## 一句话总结

> **SubAgent** = 单独叫一个助手，查完回来汇报，不沟通  
> **Agent Teams** = 组一个工作小组，分工合作，随时沟通（吵架）

---

## 用餐厅比喻理解

### 初级阶段：老板一个人忙

老板一边管经营，一边盯着后厨，忙得像"万能补位王"。

### SubAgent 阶段：叫一个帮手

老板叫「收银员」帮忙对账，收银员做完回来汇报。不沟通，只管拿结果。

### Agent Teams 阶段：分工合作，随时沟通

老板娘来了：
- 老板娘 → 负责后厨以外所有事务
- 老板 → 专注总厨总指挥

后厨团队：
- 有人备菜切菜
- 有人做主菜
- 有人做前菜
- 有人摆盘
- 有人做甜点
- 总厨检查 + 做重磅菜

**总厨和所有人随时沟通（吵架），不是做完才汇报。**

---

## SubAgent vs Agent Teams

### SubAgent 像「单独叫一个助手」

老师说：「你去图书馆帮我查资料，查完回来告诉我。」

- 一个助手
- 做完回来汇报
- 中间不沟通
- 适合：跑腿、查信息、单独任务

### Agent Team 像「直接组一个工作小组」

老师说：「你们几个人组成活动小组，自己分工合作，把班会办好。」

- 有队长（主 Agent）
- 有队员（各自负责不同工作）
- 随时互相商量
- 彼此提醒
- 最后组长统一收尾
- 适合：复杂任务、需要多角度、需要沟通的任务

---

## Agent Team 的好处

| 好处 | 说明 |
|------|------|
| **能分工** | 大家各做各的，不用一个人全包 |
| **能同时做** | 不是排队一个一个来，而是很多事情一起推进 |
| **能互相提醒（吵架）** | 一个人没想到的，另一个人可能想到了 |
| **有人统一把关** | 不会变成一堆乱七八糟的半成品 |

---

## 什么时候适合用 Agent Teams

### 适合的场景

- 事情比较复杂
- 可以拆成几部分
- 需要不同角度一起思考
- 还需要大家互相沟通的任务

### 不适合的场景

- 任务太小、太简单
- 必须按顺序做
- 多人容易互相打架（冲突难解决）

---

## Agent Teams 的 7 个问题

### 问题 1：人多协调就变复杂

- 你做哪部分？我做哪部分？谁先谁后？最后怎么合起来？
- 人越多，越容易：有人没听清、有人重复、有人忘了自己负责什么

**解决方案**：控制团队规模，提前规划分工

---

### 问题 2：会更费「资源」

每个 AI 都像忙碌的厨师，厨师越多，食材/电/空间越费。

- Agent Team 不是「免费多开几个人」
- Token 成本是很多人忽视的关键因素

**建议**：小任务不值得开大团队

---

### 问题 3：容易出现重复劳动

- 两个同学都以为自己负责买气球，买重了
- 两个人都去改同一张海报，版本冲突

**解决方案**：职责边界划分清楚

---

### 问题 4：沟通不够顺容易误会

总厨说「做清爽一点的前菜」，厨师理解成水果沙拉，总厨想要的是海鲜冷盘。

**解决方案**：前期规划阶段，把任务分工说清楚

---

### 问题 5：不一定适合小任务

如果只是「把桌子擦一下」，不需要开团队会议，一个人反而更快。

**建议**：Agent Team 更适合复杂任务，不适合小到一下子就做完的事

---

### 问题 6：如果大家都碰同一个地方，会乱

三个人同时拿着彩笔在同一个标题上改来改去，最后越改越乱。

**解决方案**：单一会话，或单独的 SubAgent 更有效果

---

### 问题 7：团队强，但不代表永远比单人聪明

一个经验丰富、目标明确的人，自己就能快速做完。组团队未必更好。

**结论**：Agent Team 不是「永远最强」，而是在「适合的任务里特别强」。

---

## 实用决策建议

> 让 AI 来帮你判断：当前任务更适合用 SubAgent 还是 Agent Teams？

| 任务类型 | 推荐方案 |
|---------|---------|
| 简单查询 / 跑腿 | 单 Agent |
| 单独任务，不需要沟通 | SubAgent |
| 复杂、需要分工、需要沟通 | Agent Teams |
| 必须按顺序执行 | 单 Agent |
| 多角度思考、多模块并行 | Agent Teams |
