## 📊 核心理念：10 分钟判断法

**核心问题**: 普通人如何在 10 分钟内判断一个 Polymarket 市场是否值得交易？

**答案**: **EV 思维 + 快速信息验证**

```
10 分钟检查清单:
□ 市场隐含概率是多少？(当前价格)
□ 我的真实概率估计是多少？(P)
□ EV = (P × 收益) - ((1-P) × 成本) 是否为正？
□ 信息源是否可靠？(多源验证)
□ 如果 P 估计偏差 10%，EV 还是正的吗？

决策:
- EV > 0 且通过检查 → 可考虑交易
- EV ≤ 0 或检查失败 → 放弃交易
```

## 🎯 学习路径设计 (1-2 周)

### 第 1 阶段：基础认知 (2-3 小时)

**目标**: 理解 Polymarket 是什么，如何赚钱

**学习内容**:
1. **Polymarket 基础** (30 分钟)
   - 什么是预测市场
   - 二元合约机制 (Yes/No)
   - 价格 = 概率 (50¢ = 50% 概率)
   - 结算机制 (正确得$1，错误得$0)

2. **赚钱逻辑** (30 分钟)
   - 低买高卖 (交易获利)
   - 持有到期 (判断正确)
   - 套利机会 (多市场价差)

3. **风险认知** (30 分钟)
   - 97% 策略死亡率 (Runes Leo 数据)
   - EV 必须为正 (唯一存活条件)
   - 概率估计偏差风险

4. **快速实践** (30 分钟)
   - 注册账户
   - 充值小额 USDC ($10-50)
   - 观察 3-5 个市场

**检查点**: 能解释"价格=概率"和"EV 计算"

---

### 第 2 阶段：EV 思维建立 (3-4 小时)

**目标**: 掌握 EV 计算和概率估计

**学习内容**:
1. **EV 公式学习** (1 小时)
   ```
   EV = (P × 收益) - ((1-P) × 成本)
   
   示例:
   - 市场价格：40¢ (隐含概率 40%)
   - 你的估计 P: 55%
   - 投入：$100
   - 收益：$150 (如果正确)
   
   EV = (0.55 × 150) - (0.45 × 100)
       = 82.5 - 45
       = $37.5 (正 EV，可交易)
   ```

2. **概率估计方法** (1 小时)
   - 基准率 (历史类似事件发生率)
   - 多源验证 (3+ 信息源)
   - 市场共识 vs 个人判断
   - 避免认知偏差

3. **Kelly 公式仓位管理** (1 小时)
   ```
   f* = (P × B - 1) / B
   
   实用建议:
   - 使用半 Kelly (降低风险)
   - 单交易不超过 2-5%
   - 避免过度集中
   ```

4. **实战练习** (1 小时)
   - 选择 5 个市场
   - 计算每个市场的 EV
   - 记录 P 估计依据
   - 模拟交易决策

**检查点**: 能独立计算 EV 并做出交易决策

---

### 第 3 阶段：信息收集与验证 (3-4 小时)

**目标**: 建立快速信息收集系统

**学习内容**:
1. **信息源分类** (1 小时)
   ```
   一级信息源 (最可靠):
   - 官方数据 (政府、机构)
   - 原始数据 (链上、API)
   
   二级信息源 (较可靠):
   - 主流媒体 (Reuters, Bloomberg)
   - 专业分析机构
   
   三级信息源 (需谨慎):
   - 社交媒体
   - KOL 观点
   - 社区讨论
   ```

2. **多源验证方法** (1 小时)
   - 交叉验证 (3+ 信息源)
   - 时间戳检查 (最新信息)
   - 利益冲突识别
   - 共识 vs 分歧分析

3. **AI 工具辅助** (1 小时)
   - Claude/GPT 信息整理
   - 自动化信息收集
   - 快速摘要生成
   - 关键信息提取

4. **实战练习** (1 小时)
   - 选择 1 个热点事件
   - 收集 5+ 信息源
   - 交叉验证
   - 形成概率估计

**检查点**: 能在 30 分钟内完成信息收集和验证

---

### 第 4 阶段：策略开发与验证 (4-5 小时)

**目标**: 开发并验证个人交易策略

**学习内容**:
1. **策略类型认知** (1 小时)
   - 价值交易 (EV 为正)
   - 套利交易 (多市场价差)
   - 信息优势 (快速反应)
   - 做市策略 (提供流动性)

2. **万次模拟方法** (1 小时)
   ```
   Runes Leo 方法:
   1. 定义策略规则 (明确入场/出场)
   2. 收集历史数据
   3. 回测模拟 (10,000+ 次)
   4. 分析存活率 (是否通过 97% 测试)
   5. 识别共同点 (如 EV 必须为正)
   6. 实盘验证 (小额开始)
   ```

3. **策略文档化** (1 小时)
   ```
   策略文档模板:
   - 策略名称
   - 入场条件 (明确规则)
   - 出场条件 (止盈/止损)
   - 仓位管理 (Kelly 比例)
   - 风险控制 (最大损失)
   - 历史回测结果
   ```

4. **实战练习** (2 小时)
   - 开发 1 个简单策略
   - 文档化
   - 模拟回测 (100+ 次)
   - 小额实盘验证 ($10-50)

**检查点**: 有 1 个文档化策略并通过模拟验证

---

### 第 5 阶段：基础设施与自动化 (2-3 小时)

**目标**: 建设稳定交易基础设施

**学习内容**:
1. **基础设施重要性** (30 分钟)
   - Runes Leo 教训："坑不在策略，在基础设施"
   - 常见问题 (API、账户、监控)
   - 预防 vs 救火

2. **基础设施检查清单** (1 小时)
   ```
   □ API 连接稳定性
   □ 多账户管理方案
   □ 余额/仓位监控系统
   □ 异常处理机制
   □ 日志记录系统
   □ 风险控制自动化
   □ 数据备份方案
   ```

3. **自动化机会** (1 小时)
   - 信息收集自动化
   - EV 计算自动化
   - 仓位计算自动化
   - 风险监控自动化
   - 日志记录自动化

4. **实战练习** (30 分钟)
   - 选择 1 个自动化点
   - 实现自动化
   - 测试验证

**检查点**: 有 1 个自动化流程在运行

## ⚠️ 常见错误与避免

### 错误 1: 忽视 EV 计算

**表现**: 凭感觉交易，不计算 EV
**后果**: 长期必亏
**避免**: 每笔交易前强制计算 EV

### 错误 2: 概率估计偏差

**表现**: 过度自信，忽视基准率
**后果**: EV 计算错误
**避免**: 多源验证，持续校准

### 错误 3: 仓位过大

**表现**: 单交易超过 10%
**后果**: 一次错误损失过大
**避免**: 使用半 Kelly，单交易≤5%

### 错误 4: 忽视基础设施

**表现**: 策略好但执行差
**后果**: 意外损失
**避免**: 先建设基础设施再交易

### 错误 5: 不记录不反思

**表现**: 不记录交易决策依据
**后果**: 无法改进
**避免**: 每笔交易记录决策表
