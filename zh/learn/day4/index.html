第4天：工具与数据工作流

预测市场不是只靠感觉。你需要一套稳定的数据和内容处理管线。

当前已具备的工具链

  • Polymarket CLI：稳定获取官方市场数据
  • xreach CLI：抓取 X / KOL 动态
  • Jina AI + RSS：新闻入口
  • defuddle：提取干净正文并写入日报
  • Cron + GitHub + Vercel：自动化发布

工具使用原则

  • 优先稳定，再追求高级
  • 优先可自动化，再追求花哨界面
  • 优先统一数据源，再扩展新源

今日任务

  • 检查一遍你的数据流是否都能单独运行
  • 确认每个环节输出文件都在预期目录
  • 把“抓取失败时的备用方案”写出来

使用建议：先读完本页，再进入对应策略页、资源页和日报案例，形成“概念 → 工具 → 策略 → 复盘”的闭环。
